# CI Nightly Smoke Report Template

Task: `TASK-010 - CI/nightly smoke plan`

This template describes the JSON shape generated by `automation/ci_nightly_smoke/generate_ci_nightly_smoke_report.py`.

The generator is a local public-safe report generator. It performs no live CI scheduling, Android, device, APK, WebView, WebRTC, payment, backend, network or production interaction.

## Command Scope

Allowed local usage:

```text
python -m automation.ci_nightly_smoke.generate_ci_nightly_smoke_report
python -m automation.ci_nightly_smoke.generate_ci_nightly_smoke_report --metadata public-safe-metadata.json
python -m automation.ci_nightly_smoke.generate_ci_nightly_smoke_report --metadata public-safe-metadata.json --output report.json
```

The metadata file is optional and must contain public-safe JSON only. It must not contain endpoints, credentials, cookies, sessions, raw logs, raw screenshots, packet captures, proxy configuration, APK artifacts, raw device identifiers, real user data, payment data, CI secrets or executable runtime/device/network recipes.

## Fail-Closed Behavior

| Input state | Expected `overall_status` | Notes |
|---|---:|---|
| No metadata file | `blocked` | Missing approval metadata blocks. |
| Missing metadata path | `blocked` | Missing file blocks. |
| Malformed JSON | `blocked` | Invalid JSON blocks. |
| Non-object JSON | `blocked` | Metadata must be a JSON object. |
| Missing prerequisite | `blocked` | Required prerequisite is not approved or not present. |
| `present` not true | `blocked` | Only literal `true` satisfies presence. |
| `evidence_status` not `confirmed` | `blocked` | Invalid status normalizes to `unknown`. |
| Complete confirmed prerequisites | `not_run` | Planned CI/nightly checks remain `not_run` and `unknown`; no successful live CI or runtime result is emitted. |

## Output Fields

| Field | Meaning |
|---|---|
| `schema_version` | Report schema version. |
| `generated_at_utc` | UTC generation timestamp. |
| `task_id` | Always `TASK-010`. |
| `mode` | Task mode declared by the generator. |
| `tool_name` | Generator identifier. |
| `overall_status` | `blocked` or `not_run`. |
| `evidence_status` | Always `unknown` for TASK-010 live CI/runtime behavior. |
| `production_safety_classification` | Always `PROD_SAFE` for local generation. |
| `redaction_status` | `redacted` when sensitive-looking values were replaced. |
| `prerequisites` | Normalized required prerequisite map. |
| `ci_jobs` | Public-safe CI job aliases and category-level metadata. |
| `planned_ci_nightly_checks` | Category-level planned checks, all `not_run` and `unknown`. |
| `blocked_reasons` | Reasons metadata cannot authorize future work. |
| `execution_notes` | Public-safe notes about local-only behavior. |
| `risks` | Public-safe risk summaries. |
| `unknowns` | Open questions with evidence status. |
| `verification` | Local report-generation verification entry. |
| `artifacts` | Public-safe redacted artifact references. |
| `review` | Reviewer placeholders. |

## Public-Safe Output Rules

Allowed:

- prerequisite names and status;
- category-level planned checks;
- CI job aliases and category-level metadata;
- blocked/not-run summaries;
- redacted artifact references;
- risk and unknown summaries;
- reviewer placeholders.

Forbidden:

- private endpoints, URLs, headers, payloads or route values;
- credentials, tokens, cookies, sessions, authorization values, API keys, CI secrets or deploy keys;
- raw device serials, IMEI/IMSI, Android IDs, MAC addresses or owner labels;
- raw logs, screenshots, videos, packet captures or dumps;
- proxy, TLS bypass or packet manipulation instructions;
- APK artifacts, source code, decompiled code, smali or method bodies;
- real user data, payment data, live CI secrets or production mutation evidence.

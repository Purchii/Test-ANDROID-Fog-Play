# Navigation Transition Report Template

Task: `TASK-011 - Navigation transition map and coverage model`

This template describes the JSON shape generated by `automation/navigation_transition_map/generate_navigation_transition_report.py`.

The generator is a local public-safe report generator. It performs no Android, device, APK, WebView, WebRTC, payment, backend, network or production interaction.

## Official Design Guidance References

The report includes public design-guideline references from:

- [Navigation on TV](https://developer.android.com/design/ui/tv/guides/foundations/navigation-on-tv);
- [TV navigation](https://developer.android.com/training/tv/get-started/navigation).

These sources are used only to categorize planned coverage for efficient, predictable and intuitive navigation; 4-way D-pad, Select, Back and Home semantics; clear paths to focusable elements; predictable Back behavior without infinite loops; and axis-based traversal for large hierarchies. They are not evidence that MTC Fog Play currently behaves this way.

## Command Scope

Allowed local usage:

```text
python -m automation.navigation_transition_map.generate_navigation_transition_report
python -m automation.navigation_transition_map.generate_navigation_transition_report --metadata public-safe-metadata.json
python -m automation.navigation_transition_map.generate_navigation_transition_report --metadata public-safe-metadata.json --output report.json
```

The metadata file is optional and must contain public-safe JSON only. It must not contain endpoints, routes, credentials, cookies, sessions, raw logs, raw screenshots, packet captures, proxy configuration, APK artifacts, raw device identifiers, real user data, payment data or executable runtime/device/network recipes.

## Fail-Closed Behavior

| Input state | Expected `overall_status` | Notes |
|---|---:|---|
| No metadata file | `blocked` | Missing approval metadata blocks. |
| Missing metadata path | `blocked` | Missing file blocks. |
| Malformed JSON | `blocked` | Invalid JSON blocks. |
| Non-object JSON | `blocked` | Metadata must be a JSON object. |
| Missing prerequisite | `blocked` | Required prerequisite is not approved or not present. |
| `present` not true | `blocked` | Only literal `true` satisfies presence. |
| `evidence_status` not `confirmed` | `blocked` | Invalid status normalizes to `unknown`. |
| Complete confirmed prerequisites | `not_run` | Planned transitions remain `not_run` and `unknown`; no successful runtime result is emitted. |

## Public-Safe Output Rules

Allowed:

- prerequisite names and status;
- category-level planned transitions;
- screen and transition aliases;
- blocked/not-run summaries;
- redacted artifact references;
- risk and unknown summaries;
- reviewer placeholders.

Forbidden:

- private endpoints, URLs, routes, headers, payloads or redirect values;
- credentials, tokens, cookies, sessions, authorization values or API keys;
- raw device serials, IMEI/IMSI, Android IDs, MAC addresses or owner labels;
- raw logs, screenshots, videos, packet captures or dumps;
- proxy, TLS bypass or packet manipulation instructions;
- APK artifacts, source code, decompiled code, smali or method bodies;
- real user data, payment data or production mutation evidence.

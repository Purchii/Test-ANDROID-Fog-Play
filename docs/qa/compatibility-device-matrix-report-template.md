# Compatibility Device Matrix Report Template

Task: `TASK-009 - Compatibility/device matrix and report format`

This template describes the JSON shape generated by `automation/compatibility_device_matrix/generate_compatibility_device_matrix_report.py`.

The generator is a local public-safe report generator. It performs no Android, device, APK, WebView, WebRTC, payment, backend, proxy, packet, network or production interaction.

## Command Scope

Allowed local usage:

```text
python -m automation.compatibility_device_matrix.generate_compatibility_device_matrix_report
python -m automation.compatibility_device_matrix.generate_compatibility_device_matrix_report --metadata public-safe-metadata.json
python -m automation.compatibility_device_matrix.generate_compatibility_device_matrix_report --metadata public-safe-metadata.json --output report.json
```

The metadata file is optional and must contain public-safe JSON only. It must not contain endpoints, credentials, cookies, sessions, raw logs, raw screenshots, packet captures, proxy configuration, APK artifacts, raw device identifiers, real user data or executable runtime/device/network recipes.

## Fail-Closed Behavior

| Input state | Expected `overall_status` | Notes |
|---|---:|---|
| No metadata file | `blocked` | Missing approval metadata blocks. |
| Missing metadata path | `blocked` | Missing file blocks. |
| Malformed JSON | `blocked` | Invalid JSON blocks. |
| Non-object JSON | `blocked` | Metadata must be a JSON object. |
| Missing prerequisite | `blocked` | Required prerequisite is not approved or not present. |
| `present` not true | `blocked` | Only literal `true` satisfies presence. |
| `evidence_status` not `confirmed` | `blocked` | Invalid status normalizes to `unknown`. |
| Complete confirmed prerequisites | `not_run` | Planned compatibility checks remain `not_run` and `unknown`; no successful runtime result is emitted. |

## Device Profile Template

Each device profile uses this shape:

```json
{
  "alias": "device-template-001",
  "target_category": "unknown",
  "os_api_band": "unknown",
  "form_factor": "unknown",
  "display_class": "unknown",
  "input_class": "unknown",
  "network_class": "unknown",
  "locale_class": "unknown",
  "evidence_status": "unknown",
  "notes": "Template only; no device inventory was collected by TASK-009."
}
```

## Planned Check Template

Each planned compatibility check uses this shape:

```json
{
  "id": "CDM-001",
  "category": "os_api_band",
  "purpose": "Public-safe category-level purpose.",
  "risk_level": "R1",
  "result": "not_run",
  "evidence_status": "unknown",
  "production_safety_classification": "PROD_SAFE",
  "execution_status": "blocked_until_approved_future_task",
  "artifact_refs": [],
  "notes": "Plan only; no compatibility execution was performed."
}
```

## Public-Safe Output Rules

Allowed:

- prerequisite names and status;
- device aliases and category-level profile fields;
- category-level planned checks;
- blocked/not-run summaries;
- redacted artifact references;
- risk and unknown summaries;
- reviewer placeholders.

Forbidden:

- private endpoints, URLs, headers, payloads or route values;
- credentials, tokens, cookies, sessions, authorization values or API keys;
- raw device serials, IMEI/IMSI, Android IDs, MAC addresses or owner labels;
- raw logs, screenshots, videos, packet captures or dumps;
- proxy, TLS bypass or packet manipulation instructions;
- APK artifacts, source code, decompiled code, smali or method bodies;
- real user data, payment data or production mutation evidence.

## Known verification rules

- `git status --short --branch` is required for every task.
- Docs-only tasks require diff review and secret/private endpoint scan by Security reviewer.
- Python automation tasks should run `python -m pytest` if test framework exists.
- Android runtime tasks require approved device/APK/config.
- Missing runtime dependencies must produce `blocked` report, not fake pass.
- Exported component guard skeletons must not publish raw component inventories or executable runtime/device recipes.
- Release gate generators must fail closed; R0/R1 gates require `status=pass` and `evidence_status=confirmed`.
- Manual runtime screen/focus map generators must fail closed; all prerequisite gates require `present=true` and `evidence_status=confirmed`, while template observations remain `not_run`/`unknown`.
- Fixture contracts must fail closed; future fixture use requires confirmed owner approval, bounded scope, synthetic-only data, resource budget, redaction, ignored evidence storage and cleanup/rollback.

- Navigation transition map generators must fail closed; future runtime transition execution requires confirmed build, target, config, transition scope, input event policy, fixture policy, resource budget, redaction, ignored evidence storage, cleanup/rollback and reviewer approvals.

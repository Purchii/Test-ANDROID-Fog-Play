# WebView Payment Safe Report Template

Task: `TASK-008 - WebView/payment safe QA plan`

This template describes the JSON shape generated by `automation/webview_payment_safe_runner/generate_webview_payment_safe_report.py`.

The generator is a local public-safe report generator. It performs no Android, device, APK, WebView, browser, redirect, payment, backend, network or production interaction.

## Command Scope

Allowed local usage:

```text
python -m automation.webview_payment_safe_runner.generate_webview_payment_safe_report
python -m automation.webview_payment_safe_runner.generate_webview_payment_safe_report --metadata public-safe-metadata.json
python -m automation.webview_payment_safe_runner.generate_webview_payment_safe_report --metadata public-safe-metadata.json --output report.json
```

The metadata file is optional and must contain public-safe JSON only. It must not contain private URLs, redirect chains, endpoints, credentials, cookies, sessions, payment data, raw logs, raw screenshots, APK artifacts, real user data or executable runtime/device/network recipes.

## Fail-Closed Behavior

| Input state | Expected `overall_status` | Notes |
|---|---:|---|
| No metadata file | `blocked` | Missing approval metadata blocks. |
| Missing metadata path | `blocked` | Missing file blocks. |
| Malformed JSON | `blocked` | Invalid JSON blocks. |
| Non-object JSON | `blocked` | Metadata must be a JSON object. |
| Missing prerequisite | `blocked` | Required prerequisite is not approved or not present. |
| `present` not true | `blocked` | Only literal `true` satisfies presence. |
| `evidence_status` not `confirmed` | `blocked` | Invalid status normalizes to `unknown`. |
| Complete confirmed prerequisites | `not_run` | Planned checks remain `not_run` and `unknown`; no successful runtime or payment result is emitted. |

## Output Fields

| Field | Meaning |
|---|---|
| `schema_version` | Report schema version. |
| `generated_at_utc` | UTC generation timestamp. |
| `task_id` | Always `TASK-008`. |
| `mode` | Task mode declared by the generator. |
| `tool_name` | Generator identifier. |
| `overall_status` | `blocked` or `not_run`. |
| `evidence_status` | Always `unknown` for TASK-008 runtime behavior. |
| `production_safety_classification` | Always `PROD_SAFE` for local generation. |
| `redaction_status` | `redacted` when sensitive-looking values were replaced. |
| `prerequisites` | Normalized required prerequisite map. |
| `flow_aliases` | Public-safe WebView/payment flow aliases and category-level fixture metadata. |
| `planned_webview_payment_checks` | Category-level planned checks, all `not_run` and `unknown`. |
| `blocked_reasons` | Reasons metadata cannot authorize future work. |
| `execution_notes` | Public-safe notes about local-only behavior. |
| `risks` | Public-safe risk summaries. |
| `unknowns` | Open questions with evidence status. |
| `verification` | Local report-generation verification entry. |
| `artifacts` | Public-safe redacted artifact references. |
| `review` | Reviewer placeholders. |

## Public-Safe Output Rules

Allowed:

- prerequisite names and status;
- category-level planned checks;
- blocked/not-run summaries;
- redacted artifact references;
- risk and unknown summaries;
- reviewer placeholders.

Forbidden:

- private URLs, redirect chains, endpoints, headers, payloads or route values;
- credentials, tokens, cookies, sessions, authorization values or API keys;
- card, wallet, bank, billing token, receipt or real payment values;
- raw WebView logs, screenshots, videos, packet captures or dumps;
- proxy, TLS bypass or packet manipulation instructions;
- APK artifacts, source code, decompiled code, smali or method bodies;
- real user data, real payment data or production mutation evidence.
